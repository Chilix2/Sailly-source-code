# TTS TTFB Instrumentation Implementation

## Overview

This implementation adds proper instrumentation to measure **Time-To-First-Audio (TTFB)** and aligns perceived vs measured latency in the system. The key issue addressed: previously, only brain processing latency was measured (2388ms), but the actual perceived latency includes TTS synthesis time and network delay.

## Problem Statement

- **Measured latency** (2388ms) only covered brain processing time: `stt_final → first text to TTS`
- **Missing components**: TTS synthesis time, network delay, WebSocket playback
- **Database issue**: `tts_first_byte_ms` column was always NULL because `_turn_timings` was never instantiated in production
- **Gap**: Needed to measure end-to-end latency from STT final to first audio byte reaching the client

### 2. Turn Timings Enhancement

**File**: `/home/charles2/sailly-browser-demo/server/brain/contracts/turn_timings.py`

#### Added new method:
```python
def tts_ttfb_ms(self) -> int:
    """Time-to-first-audio (TTFB) from brain start to first audio byte.
    
    This measures end-to-end latency from when the brain starts processing
    (after STT final) to when the first audio chunk is sent to the client.
    Includes: LLM processing + TTS synthesis + network delay.
    """
    if not self.tts_first_byte_at or not self.stt_done_at:
        return 0
    return max(0, int((self.tts_first_byte_at - self.stt_done_at) * 1000))
```

#### Updated `to_metrics_dict()`:
Now includes both metrics:
- `tts_first_byte_ms` - Time from l2_done to first audio (TTS synthesis + network)
- `tts_ttfb_ms` - Time from stt_done to first audio (entire brain + TTS)

### 3. Turn Processor Initialization

**File**: `/home/charles2/sailly-browser-demo/server/brain/v4_turn_processor.py`

Added initialization at the start of `process_turn()`:
```python
from server.brain.contracts.turn_timings import TurnTimings

# Initialize _turn_timings at the start of each turn for TTS TTFB instrumentation.
# This must happen at the top of process_turn before any async work begins.
```

This ensures that:

- TTSTimingProcessor can stamp `tts_first_byte_at` when first audio chunk is sent
- All stage timestamps are captured (stt_done_at, extract_done_at, l2_done_at, tool_done_at)

### 4. Metrics Collection

**File**: `/home/charles2/sailly-browser-demo/server/brain_service.py`

Refactored metrics dict building to properly merge TurnTimings data:

```python

_metrics_dict = { ... }

# Phase 9 A1 — per-stage latency, token counts, and cost_eur from TurnTimings.
# Merge TurnTimings metrics (which include tts_ttfb_ms computed from timestamps).
```

The `_build_turn_metrics_extra()` function calls `timings.to_metrics_dict()` which now includes both timing metrics.

## Key Metrics

### Before
- Only measured: `llm_latency_ms` = 2388ms (brain processing only)
- Missing: actual perceived latency (TTS + network)

### After
- `brain_processing_ms` (aka `llm_latency_ms`): STT final → first TTS text
- `tts_ttfb_ms`: STT final → first audio byte (perceived latency)
- `tts_first_byte_ms`: l2_done → first audio (synthesis + network only)
- `tts_synthesis_latency` (derived): tts_ttfb_ms - brain_processing_ms
