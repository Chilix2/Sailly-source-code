```markdown
## Latency Breakdown

- **Brain Processing** (`llm_latency_ms`): Time from STT final to first TTS text generated by LLM
- **TTS TTFB** (`tts_ttfb_ms`): End-to-end time from STT final to first audio byte sent to client
  - Includes: LLM processing + TTS synthesis + network delay
  - Perceived latency by user ≈ TTS TTFB
- **TTS Synthesis** = TTS TTFB - Brain Processing (synthesis + network)
```

## Architecture: How It All Connects

```
v4_turn_processor.process_turn()
    ├─ Creates: state._turn_timings = TurnTimings()
    ├─ Call: await process_turn_v4(...)
    │   └─ Records: extract_done_at, l2_done_at, tool_done_at
    └─ Return: result
    
brain_service: on_transport_output()
    ├─ Detects: LLMTextFrame (first chunk)
    │   └─ Records: timer.mark("tts_first_chunk")
    ├─ Calls: turn_processor.process_turn()
    └─ After: process TTS output...
    
Pipecat Pipeline
    ├─ TTS Service (Gemini)
    │   └─ Synthesizes audio
    └─ TTSTimingProcessor (Observer)
        └─ On TTSAudioRawFrame: stamp_tts_first_byte()
            └─ Sets: state._turn_timings.tts_first_byte_at = time.monotonic()

brain_service: finalize()
    ├─ Read: _build_turn_metrics_extra(state)
    │   └─ Calls: timings.to_metrics_dict()
    │       └─ Computes: tts_ttfb_ms = tts_first_byte_at - stt_done_at
    └─ Write: INSERT INTO google_turn_metrics(tts_ttfb_ms, ...)
```

This ensures every turn gets proper TTFB instrumentation!
