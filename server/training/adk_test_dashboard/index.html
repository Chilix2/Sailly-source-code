Side by side: ADK runner vs Current loop (Tier‑2 hybrid)

Same audio harness for both. Phase‑2 validation uses the same pipeline: scripted or GPT‑generated caller utterance → Google TTS → Deepgram STT (WER gate) → Gemini → Chirp3 bot TTS. What changes is only how Gemini is instructed and how tools are chosen or corrected after the model speaks. Run A/B without --adk-only to score both arms on identical scenarios.

ADK runner (node graph, “LLM for language, code for state”)

  • Class: ADKRunner extends ConversationLoop but overrides the bot turn.
  • Prompt: Small per‑node prompts from conversation_nodes.py (greeting, ordering, reservation, …).
  • Tools: 2–3 tools visible per node; NodeManager picks the active node from ConversationState.
  • Hybrid layer: check_forced_commits injects [TOOL:…] when state is ready (orders, reservations, dates, address, goodbye).
  • Memory: MemoryManager compresses recent context for long calls.
  • Default in this dashboard: --adk-only (table column “ADK”).

Current loop (monolithic Tier‑2 + code guardrails)

  • Class: ConversationLoop with Tier2AudioRunner._call_gemini_lm.
  • Prompt: One long system prompt in tier2_runner.py (~74 lines, many tools at once).
  • Tools: Full tool surface every turn; model must choose among all declarations.
  • Hybrid layer: _ensure_tool_call heuristics + whitelist (e.g. block create_order without dish) + auto get_menu / forced order commit when state matches.
  • Training: Same stack powers audio_training_loop / prompt optimizer workflows on Tier‑2.
  • A/B: Omit --adk-only to run Current after ADK and compare pass rate and composite.

Loop flow (one turn)

The numbered steps are the same for ADK and Current; the highlighted step is where the two architectures diverge.

Shared audio loop (both)
1
Caller text — scripted turn or _generate_caller_turn (OpenAI).
2
Caller TTS — Linear16 8 kHz (Google).
3
STT — Deepgram Nova‑3 DE, WER gate (fallback: raw text).
4
State from utterance — update_state_from_utterance.
5
Bot brain (differs) — see right column.
6
Parse tools — [TOOL:name] tags; post‑filters / dedup.
7
Bot TTS — Chirp3; update GPT history; check end_call / max turns.
Step 5 split: ADK vs Current
A
ADK: NodeManager.select_node → active node prompt + tools → Gemini → check_forced_commits → optional MemoryManager.
B
Current: Tier‑2 system prompt + full tools → Gemini → _ensure_tool_call → extra state hooks (e.g. auto get_menu, order commit).

Files: adk_runner.py, node_manager.py, conversation_nodes.py vs conversation_loop.py, tier2_runner.py.
